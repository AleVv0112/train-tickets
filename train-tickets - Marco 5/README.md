# Train Tickets (base)

Simple React + Vite app for purchasing and listing train tickets using Supabase.

Steps to run:

1. Copy `.env.example` to `.env` in the project root and set your Supabase URL and anon key:

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_KEY=your-anon-key
```

2. Install and run:

```bash
npm install
npm run dev
```

3. Database setup (run in Supabase SQL editor):

````sql
```sql
create extension if not exists "uuid-ossp";
create table if not exists tickets (
  id uuid default uuid_generate_v4() primary key,
  user_id uuid references auth.users on delete cascade,
  nombre_completo varchar(150) not null,
  horario_tren varchar(100) not null,
  destino_ruta varchar(150) not null,
  estacion_salida varchar(150) not null,
  monto_pagado decimal(10,2) not null,
  created_at timestamp default now()
);
````

Notes:

- The app uses Supabase Auth for email/password registration and sign-in.
- Replace `/info.jpg` in the project root with your info image (the app shows it on the purchase page).
- Table `tickets` must exist for the app to insert/select tickets.

## Netlify deployment and serverless verify function

This project can be deployed to Netlify and use a serverless function to securely validate tickets (so you don't expose the Supabase service role on the client).

1. Environment variables to set in Netlify (Site → Settings → Build & deploy → Environment):

- `VITE_SUPABASE_URL` = your Supabase URL (for client)
- `VITE_SUPABASE_KEY` = your anon/public key (for client)
- `NETLIFY_SUPABASE_URL` = your Supabase URL (for function)
- `NETLIFY_SUPABASE_SERVICE_ROLE_KEY` = your Supabase service_role key (for function)

2. Files added for Netlify:

- `netlify.toml` — build/publish config and SPA redirect
- `netlify/functions/verify.js` — serverless function that reads/marks tickets using the service_role key

3. How it works:

- QR generated by the app contains the full verify URL: `https://<your-site>.netlify.app/validate?id=<ID>`
- When opened, the SPA page `/validate` calls `/.netlify/functions/verify?id=<ID>` to fetch and show ticket status
- If you want the check to mark the ticket as used automatically, call `/.netlify/functions/verify?id=<ID>&markUsed=1`

4. Deploy steps (automatic):

- Push repo to GitHub.
- On Netlify: "New site from Git" → connect repo → set build command `npm run build`, publish `dist`.
- Add environment variables listed above.

5. Quick local test of function (optional):

- Install Netlify CLI: `npm install -g netlify-cli`
- Run functions locally: `netlify dev` (it will read your environment variables)

6. Required DB change:

- Add `used` column if missing:

```sql
ALTER TABLE public.tickets ADD COLUMN IF NOT EXISTS used boolean DEFAULT false;
```
